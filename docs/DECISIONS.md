# PoEnhance Decision Log

This lightweight log records engineering decisions that shape PoEnhance. It should be updated when major requirements, architecture, compliance, or product choices change.

## Decision Statuses

- Accepted: agreed and currently active.
- Provisional: selected for now, but requiring validation.
- Deferred: intentionally postponed.
- Rejected: considered and not selected.
- Superseded: replaced by a newer decision.

### DEC-046 amendment — explicit Load More

The connected Price Checker session retains the successful Search query ID, ordered result IDs, fetched-ID set, and displayed offers only in the App layer. This amendment supersedes the earlier DEC-046 deferral of additional batches and Load More. The initial Search remains one POST plus one Fetch of at most 10 IDs. When more IDs remain, a user-triggered Load More reuses that query ID and fetches only the next batch of at most 10 IDs; it never repeats Search. Failures retain the displayed offers and remaining IDs for a later explicit retry, while a new Search, Ctrl+D item, or window close cancels and invalidates the prior session. Automatic loading, queues, waiting, cache, history, refresh, and retries remain deferred.

### DEC-045 amendment — responsive first visual-layout pass

The Price Checker remains the same single full-client-height, horizontally resizable WPF tool window. Its responsive default is now `clamp(client width * 0.30, 360 DIP, 560 DIP)`, which reaches 560 DIP on a typical 1920-DIP client; persisted user width, the approximately 320-DIP user minimum, placement, and left-edge resize behavior remain unchanged. The compact presentation combines item and base names, uses official cached Trade metadata for Item Category, supports the real Category/Exact Base criterion toggle, and immediately reflects deterministic Exact Base requirements from selected components. It also shows optional already-parsed socket/link metadata, selectable modifier-component rows, inactive Min/Max and Advanced placeholders, and results in the remaining height. Modifier-value serialization, Search, Fetch, cancellation, and Load More behavior are unchanged.

### DEC-046 amendment — fixed Mirage session league

The provisional visible League text box is removed from the connected Price Checker flow. Its existing session-level league source is fixed to the existing `DefaultLeagueIdentifier` value, `Mirage`; saved or legacy UI values cannot alter Search. This does not change the Search request shape, Fetch, cancellation, diagnostics, or Load More behavior. League selection remains deferred to a future Settings decision.

Kompaktowa prezentacja łączy nazwę przedmiotu i bazy, używa oficjalnych buforowanych metadanych Trade dla Item Category, przełącza rzeczywiste kryterium Category/Exact Base i natychmiast odzwierciedla deterministyczne wymaganie Exact Base wynikające z wybranych komponentów. Pokazuje też opcjonalne już sparsowane metadane sockets/links, wybieralne wiersze komponentów modyfikatorów, nieaktywne pola Min/Max i Advanced oraz wyniki w pozostałej wysokości. Serializacja wartości modyfikatorów oraz zachowanie Search, Fetch, cancellation i Load More pozostają bez zmian.

### Uzupełnienie DEC-046 — stała liga sesji Mirage

Tymczasowe widoczne pole League usunięto z połączonego przepływu Price Checkera. Źródło ligi na poziomie sesji ma stałą wartość istniejącego `DefaultLeagueIdentifier`, czyli `Mirage`; zapisane lub starsze wartości UI nie mogą zmienić Search. Nie zmienia to kształtu requestu Search, Fetch, cancellation, diagnostics ani zachowania Load More. Wybór ligi pozostaje odłożony do przyszłej decyzji Settings.
